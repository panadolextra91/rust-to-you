# Phase 7 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

This phase hardens local resource hygiene (temp-leak / disk-exhaustion DoS) and the
safety of the startup sweep. The register below was authored at plan time (both PLAN.md
files carried `<threat_model>` blocks) and verified against the implementation — most
mitigations were exercised end-to-end during UAT (see `07-UAT.md`).

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| process → filesystem (`env::temp_dir()`) | The startup sweep enumerates and deletes dirs under the system temp dir; only the `rust-to-you-clone-` prefix is in scope | local fs entries (medium) |
| OS signal → process | SIGINT/SIGTERM/SIGHUP delivered to the binary; handled on `ctrlc`'s dedicated thread | OS signal (high — exit path) |
| panic runtime → cleanup | A panic triggers the hook before unwind; `Drop` is the redundant second pass | local resource (high) |
| `RepoMetadata.size` / clone → live-temp slot | `clone_repo` registers exactly one live temp path before `git2::clone`; cleared on `Drop` | local path (medium) |

**Structural property:** the signal callback runs on `ctrlc`'s dedicated thread (NOT raw
async-signal context), so `fs::remove_dir_all` + `eprintln!` are safe — the async-signal-safety
minefield is avoided by construction. No new dependency runs in the signal path beyond `ctrlc`.

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-07-01 | Tampering / DoS | `sweep_orphans` over-broad deletion | mitigate | Strict `starts_with("rust-to-you-clone-")` + `is_dir()` + `age >= ORPHAN_MAX_AGE` gate; only inside `env::temp_dir()`; never resolves user-supplied paths. Verified live (UAT-4: non-matching + fresh dirs survived) + unit. | closed |
| T-07-02 | DoS | Sweep deletes a concurrently-running instance's live temp dir | mitigate | Age-based liveness (D-01): a live clone keeps mtime fresh (git writes continuously) so it is never ≥60 min old while active; `duration_since` Err (clock skew) → leave the dir. Verified live (UAT-4: fresh matching dir kept). Residual: see AR-07-01. | closed |
| T-07-03 | Tampering | Symlink / TOCTOU in temp dir | accept | Names are tool-generated by `tempfile`; sweep acts only on prefix-matched same-name dirs in the per-user system temp dir. Low risk on a per-user temp. See AR-07-02. | closed |
| T-07-04 | DoS (deadlock) | Signal handler doing async-signal-unsafe work | mitigate | `ctrlc` (feature `termination`) runs the callback on a dedicated thread, not async-signal context — `fs`/`eprintln!`/alloc are safe. No raw `libc` in-handler fs. Verified: 70 unit tests + interrupt integration tests green. | closed |
| T-07-05 | Tampering | Double cleanup (panic hook before unwind, then `Drop`) leaving spurious errors | mitigate | `cleanup_live_temp()` is idempotent via `.take()` under a poison-resilient mutex; `remove_dir_all` result ignored. Verified by unit test `cleanup_idempotent`. | closed |
| T-07-06 | DoS (disk exhaustion) | Signal/panic during `collect()` leaks the live clone temp dir | mitigate | `register_live_temp` set before `git2::clone`; handler + hook call idempotent `cleanup_live_temp()`. Verified LIVE (UAT-1: SIGINT mid-clone of a 6 GB repo → temp cleaned, exit 130, no orphan) + `tests/interrupt.rs` (SIGINT+SIGTERM). | closed |
| T-07-07 | Tampering | A new `process::exit` inside a live-workspace window would reopen the leak | mitigate | D-08 invariant: all existing `process::exit` calls run post-`collect()` (workspace already Dropped); the in-handler `exit(130)` cleans first. Code-confirmed in `main.rs`. | closed |
| T-07-08 | DoS | Integration test deletes a developer's unrelated concurrent `rust-to-you-clone-*` dir | mitigate | `tests/interrupt.rs` snapshots the pre-existing `rust-to-you-clone-*` set and asserts only the child's NEW dir; never wipes the prefix broadly. Confirmed preserved through the post-review test rewrite. | closed |
| T-07-SC | Tampering (supply chain) | `ctrlc` + `libc` dependencies | accept | `ctrlc` (Detegr/rust-ctrlc, ~99.9M downloads) verified via crates.io; `libc` is the canonical rust-lang std FFI crate (dev-only, not in the release binary). slopcheck only queries PyPI — its Rust verdicts are cross-ecosystem false positives, disregarded. See AR-07-03. | closed |

*Status: open · closed*
*Disposition: mitigate (implementation required) · accept (documented risk) · transfer (third-party)*
