<objective>
Scaffold the Cargo project and build the input contract: the CLI surface (single positional
repo URL), the generous URL parser that normalizes every accepted form to `RepoRef { owner, repo }`,
and the centralized `IntakeError` taxonomy with tiered exit-code mapping. This plan delivers a
binary that accepts/rejects input correctly and exits with the right codes — all offline.

Purpose: Lock the single-command intake contract (INPT-01) and the clear read-only error
experience (INPT-02) as the stable foundation Phase 2 extends. The error enum defines the
network-dependent variants now so the taxonomy never changes shape later (D-03, D-04).

Output: A compiling `rust-to-you` binary with `Cargo.toml`, `src/main.rs`, `src/error.rs`,
and `src/cli/` (args + parser). The successful-parse branch prints a minimal placeholder and
exits 0 (the real `InvestigationSession` + stub line is added in 01-02).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI arg -> parser | The repo URL/shorthand is fully untrusted input crossing into the program. |
| RepoRef -> (Phase 2) filesystem workspace | owner/repo will later become path/URL components; must be sanitized at the Phase 1 boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | parse_repo_ref owner/repo extraction | mitigate | Validate each segment against an allowlist charset (alnum, dash, underscore, dot); reject "..", path separators, and control chars as MalformedRepoPath. Prevents path traversal into the eventual Phase 2 clone workspace (Task 2). |
| T-01-02 | Elevation | owner/repo used in later shell/clone | mitigate | Allowlist charset rejects shell metacharacters (no spaces, semicolons, pipes, dollar signs, backticks), so RepoRef can never carry an injection payload into Phase 2 git2/URL construction (Task 2). |
| T-01-03 | Denial of Service | parse_repo_ref input length | mitigate | Reject input longer than 2048 chars as NotAUrl before further processing (Task 2). Parser is O(n) string scanning, no backtracking regex. |
| T-01-04 | Information disclosure | not-found/private error message | accept | D-04: combined honest message never claims to distinguish private vs missing; no info leak. No network in Phase 1, so no live data to disclose. |
| T-01-05 | Tampering | Phase 1 read-only guarantee | mitigate | Phase 1 performs ZERO writes to any repo and ZERO network calls; no filesystem mutation beyond cargo artifacts under /target. Verified by absence of reqwest/git2/fs-write deps (Task 1, Task 2). |
| T-01-SC | Tampering | cargo dependency installs (clap, optional thiserror/url) | mitigate | clap is the STACK.md recommended crate; thiserror and url are mainstream, high-download crates.io packages. No [ASSUMED]/[SUS]/[SLOP] packages — no legitimacy checkpoint required. |
</threat_model>
