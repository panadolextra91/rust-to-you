    - prefix rule = uppercase the repo name, then FILTER to keep only [A-Z0-9] chars, THEN truncate

        edge case: 01-01's parser admits '.' in repo names, common in the Rust ecosystem —
        docs.rs, crates.io)
      - repo "crates.io" -> "CRATESIO" -> "CRAT"
      - empty-prefix fallback: if filtering leaves ZERO alphanumeric chars (a pathological repo
        name with no [A-Z0-9], e.g. "..") use the fixed placeholder "REPO" so the format still holds
    - hashing input is the literal "owner/repo" string (e.g. "tokio-rs/axum"), not owner alone
    - no randomness, no system time, no env input anywhere in case_id
    InvestigationSession::new(repo):
    - sets session.repo to the passed RepoRef
    - sets session.case_id to case_id(repo.owner, repo.repo)
    - sets started_at to now (time is allowed HERE but must not influence case_id)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RepoRef -> InvestigationSession | RepoRef is already sanitized in 01-01; session construction adds no new untrusted input. |
| run(session) -> stdout | Only the validated owner/repo and a deterministic case_id are printed; no external data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | run(session) read-only guarantee | mitigate | run() in Phase 1 only prints to stdout; no filesystem writes, no network. Verified by absence of reqwest/git2/fs::write in src/app/ (Task 2). The read-only contract is the seam invariant Phase 2 must preserve. |
| T-01-07 | Spoofing/Repudiation | case_id collisions | accept | case_id is a 16-bit hash suffix — collisions are possible but case_id is a cosmetic case label, not a security identifier; the owner/repo prefix and printed slug disambiguate. Determinism (D-08) matters more than uniqueness here. |
| T-01-08 | Information disclosure | started_at / output | accept | Only the user's own input repo slug, a derived id, and a local timestamp are printed; no secrets, no remote data (Phase 1 is offline). |
| T-01-SC | Tampering | optional chrono dependency | mitigate | chrono is the STACK.md-listed, mainstream date/time crate; high download count on crates.io. Only added if richer timestamps are wanted. No [ASSUMED]/[SUS]/[SLOP] packages — no legitimacy checkpoint required. |
