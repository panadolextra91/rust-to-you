<objective>
Implement the git-history collectors that turn the local clone into report-ready evidence (COLL-01 contributors, COLL-02 commits/branches/archaeology). Cheap full-history aggregations (total commits, contributors, bus factor, branch enumeration + tip dates) walk the whole history because they never diff (D-06). Expensive passes (most-modified-file, time-of-day) are bounded to the last 1000 commits and carry a "based on last N commits" caveat (D-05). All author metrics share ONE bot/merge/identity filter (METRICS.md). The fresh-clone branch topology trap is encoded explicitly: enumerate `BranchType::Remote`, not `Local`, or every repo reports one branch.

Purpose: COLL-01 (contributors) + COLL-02 (commits, branches, archaeology evidence). This plan extends the `repo/history.rs` started in 02-01 and adds `repo/branches.rs`, reusing 02-01's `make_fixture_repo()` test helper.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cloned repo content → collectors | untrusted git objects (commit metadata, file paths, .mailmap) are read |
| commit author strings → identity map | attacker-controlled author names/emails could spoof identity or inject `[bot]` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Elevation | reading commit/diff objects | accept | All git2 reads are metadata/tree reads; no repo file content is executed or interpreted as code; most-modified uses name-level deltas only (no content). |
| T-02-03-02 | Spoofing | author identity normalization | accept | Identity merging is best-effort (mailmap + lowercased email) for a playful report metric, not a security control; a spoofed `[bot]` suffix only changes a cosmetic count, no privilege impact. |
| T-02-03-03 | Denial of Service | unbounded history walk on huge repos | mitigate | Expensive diff pass is capped at 1000 commits (D-05, Pitfall 4); cheap counts never diff (D-06); avoids multi-second/runaway passes. |
| T-02-03-04 | Tampering | crafted file paths in diff deltas | accept | Paths are used only as HashMap keys for a counter; never opened, joined, or written; `to_string_lossy` avoids panics on non-UTF8. |
</threat_model>
