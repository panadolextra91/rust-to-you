<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git tree paths → oldest_file string | Tree entry paths come from an untrusted cloned repo (arbitrary bytes / unicode / very long paths). |
| git author times → oldest_contributor min | Author timestamps are attacker-controllable (negative, zero, far-future). |
| snapshot → pure analyzers | In-process normalized data; no external input crosses here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service | oldest_file on empty / single-empty-tree repo | mitigate | Return Ok(None) when no commits or empty intersection; no unwrap on revwalk.next() / BTreeSet first. |
| T-03-05 | Information Disclosure | path traversal via crafted tree path in oldest_file | accept | oldest_file is a display string only; never used to open/read a filesystem path; Phase 4 renders it as text. |
| T-03-06 | Denial of Service | oldest_contributor min over negative/future author times | mitigate | Use i64 comparison; no subtraction; tolerate any timestamp without panic. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies (git2/chrono/tokei already vendored in Phase 2); slopcheck N/A. |
</threat_model>
