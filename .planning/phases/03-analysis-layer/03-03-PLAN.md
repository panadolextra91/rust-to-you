<objective>
Assemble the six factual report sections into a single pure-data `FactualSections` view model
that Phase 4 will render. Five sections wrap the analyzers built in 03-01/03-02
(CommitCrimes, BranchJungle, AncientRelics, LanguageSoup, InfrastructureFootprints); the sixth,
First Impressions, is assembled here from existing snapshot metadata (age, default branch, stars,
forks, contributor count, last activity).

Purpose: Phase 4 needs one stable, pure-data entry point per the report view-model boundary
(D-06) — numbers/strings/enums/Option only, no formatting. This plan is the convergence point
that wires every analyzer into one struct and proves the whole assembly is deterministic.
Output: `src/report/` module with `FirstImpressions` + `FactualSections` structs and a
`build_factual_sections(snapshot) -> FactualSections` function, fixture-tested end-to-end.
</objective>
