<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git history → analyzer date math | Commit author times are attacker-controllable (any value, including future or negative) since they come from a cloned public repo. |
| snapshot → pure analyzers | In-process, already-normalized data; no external input crosses here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | commits_within_days date arithmetic | mitigate | Compute cutoff in i64 (now_secs - days*86400); never subtract on usize; tolerate negative/future author times without panic. |
| T-03-02 | Denial of Service | branch_jungle on empty branch list | mitigate | oldest_branch returns None and counts stay 0 when branches is empty (no unwrap on min). |
| T-03-03 | Tampering | crafted mailmap / author name in top_contributor_name | accept | Name is read-only display data; no shell/path interpolation; rendered later by Phase 4 which is responsible for sanitization. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies added this plan (git2/chrono already vendored in Phase 2); slopcheck N/A. |
</threat_model>
