<objective>
Establish the intake contract surface for Phase 6: the two new error variants with their exit
codes, the hardened parser (leading-dash rejection + segment length caps), the `--deep` clap flag,
and the `size` field on `RepoMetadata`. This is the Wave 0 test-enabler plan for these four files ‚Äî
every artifact ships with the unit tests that lock its behavior, so the downstream size-guard plan
(06-02) can consume stable contracts.

Purpose: Reject malformed/malicious input at the parser before any network or git2 call (SEC-01),
prove it with an injection/abuse test table (SEC-02), and create the `RepoTooLarge` error + `--deep`
flag + `size` field that the size guard in 06-02 depends on.

Output: `IntakeError::RepoTooLarge` (exit 6) and `IntakeError::UnsafeInput` (exit 2);
hardened `parse_repo_ref`; `Args.deep`; `RepoMetadata.size`; new `#[cfg(test)]` modules in
`error.rs` and `args.rs`; extended reject table in `parse.rs`; extended `decode()` fixtures; the
existing `RepoMetadata` test literal in `report/sections.rs` updated for the new required field.
</objective>
