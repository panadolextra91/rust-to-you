<objective>
Insert the pre-flight size guard into the existing `fetch_metadata → clone_repo` gap in
`src/app/collect.rs`, gated on the `--deep` flag threaded through `InvestigationSession`. The guard
refuses repos larger than 500 MB BEFORE the clone starts (so the machine never begins an oversized
clone), warns-and-proceeds with `--deep`, and fails open (warn + clone) when size is unknown.

Purpose: GUARD-01 (refuse-by-default oversize protection via the already-fetched metadata) and
GUARD-03 (`--deep` opt-in to full analysis of a large repo). The decision logic is extracted into a
pure `size_decision` helper so it is unit-testable without a live network or clone (the Wave 0
enabler for collect.rs).

Output: `MAX_REPO_KB` constant, `size_decision(meta_state, deep) -> SizeDecision` pure helper,
the guard branch in `collect`, `InvestigationSession.deep`, `main.rs` wiring, and a
`#[cfg(test)]` module covering all four decision branches.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RepoMetadata.size (remote) → size guard | Remote-reported size drives the refuse/proceed decision; a hostile under-report could understate true clone cost |
| collect() → clone_repo (git2 FFI) | The guard sits on the path to the only resource-heavy operation; refusal must happen before clone starts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Denial of Service | oversize-clone machine hang | mitigate | Pre-flight `size > 500 MB` refuse-by-default in collect.rs BEFORE clone_repo (GUARD-01, D-04); `--deep` is an explicit, warned opt-in (D-07); guard returns Err up to main.rs so no clone starts |
| T-06-07 | DoS (false-negative) | remote `size` under-reports (LFS-excluded, A2) | accept | `size` excludes Git LFS and may under-report; the guard is a heuristic safety net, not a hard quota — a repo just over threshold in LFS may slip past, but `--deep` and clone still proceed. Low risk; documented in RESEARCH Assumptions A2 |
| T-06-08 | Availability regression | metadata Unavailable (rate-limit / no token / flaky net) | accept (fail-open) | D-09: warn + proceed when size unknown — never block on missing data, preserving the tool's core value on a flaky API; the guard never returns Err on Unavailable |
| T-06-09 | Resource leak / EoP | process exit while CloneWorkspace alive | mitigate | RepoTooLarge returns Err BEFORE clone_repo, so no workspace is alive; no `std::process::exit` is added in collect.rs (full temp hygiene is Phase 7) |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No packages installed this phase (zero new deps) — nothing to slopcheck |
</threat_model>
