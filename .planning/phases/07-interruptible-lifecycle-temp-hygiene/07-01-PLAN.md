<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process → filesystem (`env::temp_dir()`) | The sweep enumerates and deletes directories under the system temp dir; only the `rust-to-you-clone-` prefix is in scope. |
| OS signal → process | SIGINT/SIGTERM/SIGHUP delivered to the running binary; handled on ctrlc's dedicated thread. |
| panic runtime → cleanup | A panic triggers the hook before unwind; Drop is the redundant second pass. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering / DoS | `sweep_orphans` over-broad deletion | mitigate | Strict `name.starts_with("rust-to-you-clone-")` + `is_dir()` + `age >= ORPHAN_MAX_AGE` gate; only inside `env::temp_dir()`; never resolves user-supplied paths (ASVS V12). Unit-tested: non-matching dirs survive. |
| T-07-02 | DoS | Sweep deletes a concurrently-running instance's live temp dir | mitigate | Age-based liveness (D-01): a live clone keeps mtime fresh (git writes continuously), so it is never ≥ 60 min old while active. `duration_since` Err (clock skew) → leave the dir. Residual risk accepted: a paused/suspended instance >60 min could be swept — vanishingly rare for a single-shot CLI; documented. |
| T-07-03 | Tampering | Symlink / TOCTOU in temp dir | accept | Names are tool-generated by `tempfile`; sweep only acts on prefix-matched same-name dirs in the per-user system temp dir. Low risk on a per-user temp; not engineered around. |
| T-07-04 | DoS (deadlock) | Signal handler doing unsafe work | mitigate | `ctrlc` runs the callback on a dedicated thread, not async-signal context — `fs`/`eprintln!`/alloc are safe (avoids the raw-libc async-signal-safety minefield). |
| T-07-05 | Tampering | Double cleanup (panic hook before unwind, then Drop) leaving spurious errors | mitigate | `cleanup_live_temp()` idempotent via `.take()` under the mutex; `remove_dir_all` result ignored. Unit-tested by `cleanup_idempotent`. |
| T-07-SC | Tampering | `ctrlc` / `libc` supply chain | accept | Both well-known crates.io packages verified via crates.io API (ctrlc 99.9M dls, source github.com/Detegr/rust-ctrlc; libc is the std FFI crate). slopcheck only queries PyPI so its `[SLOP]`/`[SUS]` verdicts on these Rust crate names are cross-ecosystem false positives — disregarded per RESEARCH Package Legitimacy Audit. |
</threat_model>
