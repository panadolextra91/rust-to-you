interrupt guarantee end-to-end. `CloneWorkspace` registers its temp path on create and clears the

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS signal → process | SIGINT/SIGTERM mid-run cross into the ctrlc handler thread; must clean the live temp before exit. |
| process → filesystem | `clone_repo` creates a temp dir under `env::temp_dir()`; the global slot tracks exactly one live path. |
| panic runtime → cleanup | A panic during `collect()` triggers the hook; Drop is the redundant pass under unwind. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | DoS (disk exhaustion) | Signal/panic during `collect()` leaks the live clone temp dir | mitigate | `register_live_temp` set before `git2::clone`; handler + hook call idempotent `cleanup_live_temp()`; `tests/interrupt.rs` proves no orphan after SIGINT/SIGTERM (CLEAN-01). |
| T-07-07 | Tampering | A new `process::exit` added inside a live-workspace window would reopen the leak | mitigate | D-08 invariant: existing exits all run post-`collect()` (workspace Dropped); plan forbids new in-window exits. Reviewed in Task 2. |
| T-07-08 | DoS | Integration test deletes a developer's unrelated concurrent `rust-to-you-clone-*` dir | mitigate | Test snapshots pre-existing `rust-to-you-clone-*` set and asserts only the child's new dir; never wipes the prefix broadly. |
| T-07-SC | Tampering | `libc` dev-dependency supply chain | accept | `libc` is the canonical Rust std FFI crate (rust-lang org); dev-only, not shipped in the release binary. slopcheck PyPI verdicts disregarded (crates.io is authoritative — see RESEARCH Package Legitimacy Audit). |
</threat_model>
