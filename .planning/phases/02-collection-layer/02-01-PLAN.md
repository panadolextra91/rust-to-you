<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| input URL → clone target | `RepoRef` (Phase-1 validated) is interpolated into the clone URL; untrusted owner/repo could attempt path/URL injection |
| remote repo → local temp workspace | cloned content is untrusted; must never be executed |
| process lifecycle → temp dir | `TempDir` must drop (clean up) even on error/panic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | clone URL built from owner/repo | mitigate | Build URL only from Phase-1-validated `RepoRef` (charset-restricted, `..`/`\` already rejected in parse.rs); no other interpolation; clone target is a fresh owned `TempDir`. |
| T-02-01-02 | Elevation | cloned repo content | accept | git2 clone is read-only; we never run repo hooks or execute/parse file contents in this plan (repo-age only reads commit metadata). |
| T-02-01-03 | Information Disclosure | temp clone dir on disk | mitigate | `tempfile::TempDir` creates user-only-permission dirs and deletes on Drop; doc-comment forbids `process::exit` while a workspace is alive (Pitfall 5 — leak prevention). |
| T-02-01-04 | Information Disclosure / Tampering | HTTPS clone transport | mitigate | Clone over `https://` only; git2 (OpenSSL) validates certs by default — cert verification is never disabled. |
| T-02-01-SC | Tampering | cargo deps added this plan | mitigate | All 7 packages slopcheck-audited in 02-RESEARCH.md § Package Legitimacy Audit (5 [OK], tokei [SUS] confirmed false positive). No [ASSUMED]/[SLOP] packages → no blocking checkpoint required. |
</threat_model>
