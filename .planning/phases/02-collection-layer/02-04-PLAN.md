<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| metadata (degradable) + clone + scan → snapshot | all upstream untrusted inputs converge into one struct consumed by Phase 3 |
| cloned working tree → scan | tokei + infra read files under the clone; must not escape via symlink or execute content |
| process lifecycle → temp workspace | snapshot must be built and workspace dropped before any exit() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering / Elevation | scanning cloned files | mitigate | tokei counts lines and infra uses `Path::exists()` joins UNDER the clone root; no file content is executed/parsed-as-code; do not follow symlinks out of the workspace (symlink-escape mitigation, Security Domain). |
| T-02-04-02 | Information Disclosure | temp clone lifecycle | mitigate | `collect()` never calls `process::exit`; `CloneWorkspace` (TempDir) drops at function return; `main` maps errors to exit codes only after the workspace is gone (Pitfall 5). |
| T-02-04-03 | Denial of Service | degraded metadata aborting the run | mitigate | Transient API failure degrades to `RepoMetaState::Unavailable` instead of aborting (D-03), so one flaky call cannot sink an otherwise-valid investigation; only a 404 (abort) and a clone failure (hard) stop the run. |
| T-02-04-04 | Information Disclosure | snapshot summary output | mitigate | Summary prints repo facts only; no `GITHUB_TOKEN`/auth material is ever placed in the snapshot or printed (carried from 02-02 T-02-02-02). |
</threat_model>
