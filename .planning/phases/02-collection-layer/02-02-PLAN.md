<objective>
Build the GitHub metadata client: a single blocking REST call to `GET /repos/{owner}/{repo}` that supplies the platform facts git cannot know (stars, forks, description, topics, pushed_at). Implement the status-to-error mapping as a PURE function so it is unit-testable without a network, set the mandatory User-Agent, and optionally authenticate with `GITHUB_TOKEN` (D-04). The call follows the API-first, degrade-on-transient policy (D-02/D-03): a 404 is the abort signal that finally populates Phase 1's reserved `IntakeError::RepoNotFoundOrPrivate`; transient failures are surfaced as errors the orchestrator (02-04) will degrade rather than abort on.

Purpose: COLL-01's platform metadata, with the two verified traps encoded — User-Agent mandatory (no UA → 403) and 404→abort vs 403/network→transient. Keeping `classify` pure keeps live network out of the default test suite (02-VALIDATION.md).

Output: New `src/github/` module (`mod.rs`, `client.rs`) with `RepoMetadata`, `GithubError`, pure `classify`, and `fetch_metadata`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → GitHub REST API | outbound HTTPS request to api.github.com with optional secret token |
| env (GITHUB_TOKEN) → process | secret read from environment, used as bearer auth |
| GitHub response → process | untrusted JSON body deserialized into RepoMetadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing / MITM | api.github.com transport | mitigate | HTTPS only via reqwest+rustls; certificate verification left at default (never disabled). |
| T-02-02-02 | Information Disclosure | GITHUB_TOKEN handling | mitigate | Token read from env, attached via `bearer_auth`, never printed/logged; no error path echoes the Authorization header (Security Domain V7). |
| T-02-02-03 | Denial of Service | rate-limit / no User-Agent | mitigate | Mandatory User-Agent (Pitfall 2) prevents UA-403; exactly one request (Pitfall 2); 403/429 classified as RateLimited and degraded downstream (D-03). |
| T-02-02-04 | Tampering | untrusted JSON deserialization | accept | serde into a fixed struct with typed fields; unknown fields ignored; no code execution from JSON; low-value read-only data. |
</threat_model>
