## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo-derived strings → renderer | Branch/file/contributor names + repo description (already validated in Phase 1) flow into Spans (TUI) and plain text (non-TTY). |
| process → terminal/pipe | Rendered output is written to a TTY (Task 2 path, exercised in 04-02) or a pipe (Task 3 plain path). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | repo strings rendered into TUI report (report.rs) | mitigate | Render via ratatui Span/Buffer/Cell model (Task 2) — content is written as cells, not raw bytes; never `print!` raw repo strings to the screen. |
| T-04-02 | Tampering | repo strings emitted to a pipe (plain.rs) | accept | Data fields are names/paths/booleans (low risk); `\| less` use case is acceptable. Plain renderer emits no raw escape bytes (asserted: no `\x1b[`). |
| T-04-03 | Denial of availability | gigantic Vec<Line> on a huge repo | accept | Render input is bounded (6 sections, short language list, integer branch counts — not per-branch lines). No unbounded growth. |
| T-04-SC | Tampering | cargo dependency installs (ratatui, crossterm) | mitigate | Both packages are canonical, named in STACK.md, corroborated by official docs.rs (Package Legitimacy Audit in RESEARCH = Approved, no [ASSUMED]/[SUS]). Confirm single crossterm version via `cargo tree -i crossterm`. |
